в—Ҹ No, Crawl4AI does not use embed.den.lan. The Crawl4AI service is a standalone Python FastAPI service that:

  1. Uses Playwright for headless browser automation
  2. Uses Crawl4AI library (v0.7.6) for AI-optimized web crawling
  3. Applies BM25 algorithm for markdown filtering (statistical text ranking, not embeddings)
  4. Returns processed markdown content

  The BM25 algorithm used by Crawl4AI is a keyword-based ranking function, not an embedding-based approach. It doesn't require any external embedding service.

  If you want Scrapegoat to use embed.den.lan for embedding generation, that would need to be configured separately in the main Scrapegoat service's embedding configuration (not in the Crawl4AI service).
